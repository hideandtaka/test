## コード解説スキル（初心者向け）

プログラミング初心者でも理解できるように、コードを1つずつ丁寧にやさしい日本語で解説する。

### 発動条件（自動検知）

以下のような場面で発動する：

- ユーザーがコードを貼り付けて「これ何？」「わからない」と聞いた
- 「解説して」「説明して」「教えて」「読めない」「意味は？」と言われた
- ファイルを指定して「中身を教えて」と言われた
- エラーの原因を理解するためにコードの動きを知りたい場面

### 解説のルール

#### 1. まず全体像を伝える（ひとことまとめ）

最初に「このコードはざっくり言うと○○をしています」と、一言で全体の目的を伝える。

#### 2. ブロックごとに分けて解説する

コードを意味のあるまとまり（ブロック）に分けて、それぞれ以下の形式で解説する：

```
### 📦 ブロック①：○○をしている部分（行番号）

​```言語名
（該当するコード）
​```

**何をしているか：**
やさしい日本語で説明。

**たとえるなら：**
日常生活のたとえ話で理解を助ける（適切な場合のみ）。
```

#### 3. 専門用語には必ず補足をつける

専門用語が出てきたら `関数（＝何かの処理をまとめたもの）` のようにカッコ書きで補足する。

よく出る用語の説明例：
- **変数** → 「データを入れておく箱のようなもの」
- **関数** → 「処理をひとまとめにして名前をつけたもの」
- **引数（ひきすう）** → 「関数に渡す材料のようなもの」
- **戻り値** → 「関数が処理した結果として返してくれるもの」
- **配列** → 「データを順番に並べて入れておけるリスト」
- **オブジェクト** → 「名前付きのデータをまとめたもの」
- **ループ** → 「同じ処理を何度も繰り返すこと」
- **条件分岐** → 「もし○○なら△△する、という判断のしくみ」
- **import** → 「他のファイルから機能を借りてくること」
- **export** → 「このファイルの機能を他のファイルに貸し出すこと」

#### 4. 処理の流れを番号付きで示す

最後に「このコードの処理の流れ」として、上から順に何が起きるかを番号付きリストでまとめる。

```
### 🔄 処理の流れ

1. まず○○が実行される
2. 次に△△の値をチェックする
3. もし条件に合えば□□を行う
4. 最後に結果を返す
```

#### 5. 質問を促す

解説の最後に「わからない部分があれば気軽に聞いてくださいね！」と一言添える。

### 解説のトーン

- 「〜ですね」「〜ましょう」など、やさしく語りかける口調
- 上から目線にならない
- 「簡単です」「当然ですが」のような表現は使わない（初心者にはプレッシャーになる）
- 間違いを恐れなくていいという安心感を与える

### 注意事項

- ユーザーがファイルパスを指定した場合は、Readツールでファイルを読み取ってから解説する
- 長いコードの場合は、重要な部分に絞って解説し「他の部分も見ますか？」と聞く
- ユーザーのレベルに合わせる。質問の仕方から経験値を推測し、説明の深さを調整する
