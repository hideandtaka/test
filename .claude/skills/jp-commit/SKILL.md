## 日本語コミットスキル

変更内容を分析し、日本語でわかりやすいコミットメッセージを作成してコミットします。
このスキルはコミットが必要なタイミングで自動的に発動します。

### 手順

1. `git status` と `git diff --staged` および `git diff` を実行して、変更内容を確認する
2. `git log --oneline -5` で最近のコミット履歴を確認する
3. 変更内容を分析し、以下のフォーマットで日本語コミットメッセージを作成する
4. ユーザーに確認してからコミットする

### コミットメッセージのフォーマット

```
<種別>: <変更の要約（日本語、50文字以内）>

<変更の詳細説明（日本語、必要に応じて箇条書き）>

Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
```

### 種別の一覧

| 種別 | 用途 |
|------|------|
| 機能追加 | 新しい機能の追加 |
| 修正 | バグの修正 |
| 改善 | 既存機能の改善・リファクタリング |
| ドキュメント | ドキュメントの追加・更新 |
| テスト | テストの追加・修正 |
| 設定 | 設定ファイルの変更 |
| 削除 | 不要なコードやファイルの削除 |
| スタイル | コードスタイルの変更（動作に影響なし） |
| ビルド | ビルド関連の変更 |
| CI | CI/CD関連の変更 |

### 注意事項

- ステージされていない変更がある場合は、ユーザーにどのファイルをステージするか確認する
- `.env` やクレデンシャルファイルなど機密情報を含むファイルは警告する
- コミットメッセージはHEREDOCを使って渡す
- `--no-verify` は使用しない
- pushはユーザーが明示的に指示した場合のみ行う
